# Pilot Pitch Summary — AgenticFlow v0.1.0 Pilot RC

## One-line pitch

AgenticFlow turns engineering drawing notes into governed BOQ, manufacturing, quotation, approval, audit, and export packages through a five-layer governed AI organism.

## Problem

Engineering teams lose time translating drawing notes into quantities, manufacturing plans, quotations, review packages, approvals, and audit trails. The work is repetitive, fragmented, and risky when governance is not explicit.

## Solution

AgenticFlow provides a deterministic pilot workflow that demonstrates how a governed engineering AI platform can structure drawing notes, generate package artifacts, require role-based review, and preserve audit/model-role traceability before release.

## Demo workflow

Login → Project Dashboard → Pilot Demo Checklist → Multi-Part Package Console → combined BOQ → combined manufacturing plan → combined quotation → Senior Structural Engineer approval → Package History → HTML export / Print to PDF.

## Novelty

- Five-layer governed AI organism architecture.
- Provider-agnostic AI Gateway with model-role routing.
- PatchD governance and Patch E approval controls.
- Deterministic drawing-to-BOQ-to-manufacturing-to-quotation pipeline.
- Built-in approval, audit, model-role audit, and export surfaces.

## Target users

- Structural engineering teams.
- Steel fabricators and estimators.
- Engineering managers reviewing package risk and approval state.
- Pilot customers evaluating governed AI workflows for engineering operations.

## Why Valor Struct is different

Valor Struct combines engineering domain workflows, governance, manufacturing awareness, quotation logic, and auditability in one deterministic pilot package instead of showing isolated chat prompts or ungoverned automation.

## Current MVP scope

The MVP supports a deterministic Canopy Base Plates Demo with BP-01, BP-02, and BR-01 text notes, expected package totals, local demo auth/workspace context, package approval, audit/model-role history, and export review.

## Next roadmap

- Activate PostgreSQL-backed production persistence.
- Expand supported deterministic part families after pilot validation.
- Improve package review UX and export bundles.
- Add deployment automation and operational monitoring.
